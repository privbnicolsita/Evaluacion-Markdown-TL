# Evaluación de Conocimientos: GitHub y Markdown

 

## 1. Conceptos Generales

**¿Qué es un archivo Markdown y para qué sirve?**

> Escribe tu respuesta aquí...
Es básicamente un archivo de texto donde puedes escribir cosas y darle formato sin necesidad de programas complicados. Sirve para hacer tus proyectos más claros, poner títulos, listas, enlaces, negrita, cursiva, todo de manera fácil, y que quede bonito en GitHub.
 

**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**

*Escribe tu respuesta aquí (recuerda usar formato cursiva como se pide en la evaluación).*

 El README.md es como la “presentación” de tu proyecto. Ahí cuentas de qué va, cómo usarlo y cualquier cosa importante que otros tengan que saber. Es lo primero que alguien ve cuando entra a tu repo.

## 2. Investigación de Sintaxis Markdown

Completa la siguiente guía de referencia con los símbolos correctos:

 

| Elemento | Sintaxis Markdown (Ejemplo) | Resultado Visual |

| :--- | :--- | :--- |

| **Título Nivel 1** | `# Texto` | **Título Grande** |

| **Título Nivel 2** | `## Texto` | **Título Mediano** |

| **Texto en Negrita** | `**Texto**` | **Texto** |

| **Texto en Cursiva** | `*Texto*` | *Texto* |

| **Lista Desordenada** | `- Elemento` | • Elemento |

| **Lista Ordenada** | `1. Elemento` | 1. Elemento |

| **Insertar un Enlace** | `[Nombre](URL)` | [Enlace](https://google.com) |
